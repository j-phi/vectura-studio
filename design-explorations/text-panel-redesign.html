Design thesis

Lean into surprise-and-delight while staying consistent with Vectura's engineering vibe. Signature micro-interactions: the font picker animates a tiny stroke being "drawn," the Hand-Jitter slider visibly jitters its own label, toggles have a satisfying ON/OFF snap, sections expand with the panel-slide easing, the plotter/pen metaphor shows up (a little pen-nib cursor or "pen will travel: Left→Right" affordance for plot order). Playful but never gimmicky — every flourish reinforces meaning.

Key moves

  • Four disclosure sections mirror the data flow (Content & Face -> Sizing & Placement -> Outline -> Fill), with gate toggles living in the section headers and dependent params indented one step under a dashed causality rail. All collapse via grid-template-rows 1fr->0fr at 220ms cubic-bezier(.22,1,.36,1) — the exact panel-slide token from src/ui/skin/motion.css — so the panel reads as one material.
  • Signature pen/plotter metaphor is wired everywhere: the pane header carries a pen-nib glyph, each font face draws its specimen stroke via stroke-dashoffset on hover/select (outline/Google faces additionally reveal a hatch sweep to teach 'fillable'), and Plot Order shows a traveling pen-nib affordance with live 'pen travels L -> R' vs 'follows layout order' copy.
  • Hand Jitter self-demonstrates: changing the slider fires a short, bounded wobble burst on its own label (amplitude proportional to value, 0 = perfectly still), then settles — only on value change, never on every render, so it communicates state without becoming noise.
  • Conditional logic is fully live and feel-able: Fit-to-Frame swaps Frame Fill <-> Size (with a one-shot frame-fill pulse), Outline Weight > 1 instantly reveals the Thickening Mode segmented control, and selecting a built-in single-stroke face dims Bezier/Smoothness/Fill to 40% in place with a one-line teachable reason rather than hiding them — so users learn the stroke-vs-outline model.

  • Full prefers-reduced-motion fallback collapses every transition/animation to instant opacity, and all tokens (palette, radii, spacing, --slider-start gradient, Space Grotesk + JetBrains Mono mono numerics) match the Meridian Blue Dark system exactly at the true 290px pane width.

Adversarial review

Lost functionality flagged: Fill section entirely non-interactive: the fill body is wrapped in `<div class="gated-off" id="fillGate"><div class="lock">`, and the static rule `.gated-off .lock{opacity:.4; pointer-events:none}` matches UNCONDITIONALLY (it is not scoped to a stroke-face state). So even with the default Google/Inter face active, fillType swatches, Density, Angle, and the Advanced (+18) params are all dimmed and pointer-events:none — fillEnabled can toggle the header but the controls beneath can never be clicked. Fill is unreachable for every face.; bezierOutline and Smoothness are permanently disabled: both rows carry class `lock` inside `#googleOnly` (class `gated-off`). `.gated-off .lock{opacity:.4; pointer-events:none}` applies regardless of face, so the True Bezier Curves toggle and Smoothness slider are at 40% and non-interactive even when a Google outline face is selected — the exact faces they are supposed to serve.; Plot Order is collapsed away whenever outlineStroke is OFF: it lives inside `#cond-outlineBody`, which the outlineStroke handler hides wholesale. Plot order (pen path efficiency) is independent of whether the glyph stroke is drawn, so turning off the stroke needlessly removes a reachable control.

Tokens and typography are genuinely faithful — 290px pane, panel-slide 220ms cubic-bezier(.22,1,.36,1) reused for section collapse, --slider-start #80c4f0, Space Grotesk + JetBrains Mono numerics, radii and accents all match Meridian Blue Dark, and the prefers-reduced-motion fallback is real. The information architecture (four data-flow sections, header gate toggles, dashed causality rail, teachable visible-but-disabled gating) is a strong, on-brand idea. BUT the implementation undercuts its own thesis at the two most load-bearing points: the unconditional `.gated-off .lock{opacity:.4;pointer-events:none}` rule leaves the ENTIRE Fill section and the bezier/smoothness controls permanently dimmed and unclickable for every face — including the Google faces they exist to serve — because the JS that was supposed to make the dim conditional is three empty no-op loops with a comment admitting it's 'handled elsewhere' (it isn't). So the single most-praised flourish — 'keep Google-only controls visible-but-dimmed with an inline reason so the panel teaches its model' — actually renders ~20 fill params and 2 outline params dead on arrival for the default Inter face. That is a functionality drop, not a teachable disable. Add the over-eager outlineStroke gate swallowing Plot Order, the mislabeled 0..30/÷10 jitter mapping, the guessed stroke-draw lengths that won't fully draw, and the seg-glide mispositioning inside collapsed sections, and the result reads as a beautiful, well-tokened shell whose conditional logic was never run end-to-end. The delight is mostly meaningful in concept (specimen stroke teaches stroke-vs-outline, jitter self-demos, pen-travel copy is genuinely clarifying) but the core gating is broken, so as shipped it fails the 'easiest possible' goal the delight was supposed to serve.

Design thesis

Beginner-first. The panel opens showing only the four things that matter 90% of the time — Text, Font, Size/Fit, Align — beautifully spacious. Everything else (spacing, offset, jitter, outline, fill, common) lives under a single clearly-labeled "Advanced" / sectioned disclosure that is collapsed by default. Optimize for "a new user can make great text in 10 seconds and never feels overwhelmed," while a power user is one click from everything.

Key moves

  • Hero-four block always visible at the top (Text, Font, Align, Size/Fit) with generous 14px vertical rhythm and a quiet 'Essentials' divider — a new user makes great text without ever scrolling; the four Advanced sections (Spacing & Offset, Outline, Fill, Common) sit below as flat, one-level-deep collapsed disclosures.
  • Reveal-on-intent in place: Fit-to-Frame swaps the Size slider for the Frame-Fill ratio at the same spot (animated grid-template-rows, no scroll jump); Outline toggle reveals its body; Weight>1 reveals the Thickening mode in an indented nested row; Fill toggle reveals pattern grid + params — all height-animated to avoid reflow shock.
  • Active-state dot on section headers (Linear-style): a filled accent dot with translucent halo appears whenever a collapsed section holds non-default values, so power users see 'something's set' without expanding. Wired live to slider/toggle deltas via markActive().
  • Disabled-with-reason, never hidden, for face-capability gating: on a built-in single-stroke face the Bézier/Smoothness group and the entire Fill section render dimmed (0.42 opacity, locked) with an inline dashed 'Outline (Google) face required' note — teaches that switching font unlocks power instead of making controls vanish.
  • The single most confusing thing ('fill does nothing on a built-in face') is surfaced as a persistent, quiet hint right under Font that rewrites itself per face ('Bézier & Fill unlocked' vs 'Fill does nothing here'), not buried only in a section the user may never open. Click the font picker to cycle Inter → Pacifico → Vectura Sans → Condensed and watch every dependent gate react.
  • Inspector-row spine: a fixed label-column / control-column grid (1fr / 150px) aligns every widget to one vertical edge; sliders carry inline mono tabular-num value chips with units, and the track gradient paints slider-start→accent up to the thumb — all on the exact Meridian Blue dark tokens.

A beautifully designed shell undone by a single load-time hoisting bug. The thesis — tab organization plus face-aware conditional gating with a self-disabling Fill tab and a status chip — is sound and the static composition is the most polished and token-faithful of the set. But the concept exists to PROVE its conditional logic, and on every fresh load a TypeError aborts initialization, leaving the font picker, all face gating, the redirect cue, every segmented control, the thickening reveal, and even the just-'fixed' Curves toggle completely non-functional. The changelog claims a long list of behavioral fixes that are unreachable in practice because the code never runs that far. The revision fixed the architecture on paper (CSS-driven state, dim-don't-remove, equal-height swap) but shipped a fatal init-order regression, so the panel demonstrates the opposite of completeness. High design ceiling, near-zero working conditional behavior.

Standout. The static visual design is genuinely excellent and the most on-brand of the concepts: a compact accent-blue segment bar with a sliding active fill, a persistent Face mode-chip, a sticky Common footer pinned across tabs, tasteful subheads, mono tabular numerics, and a clean dark Meridian palette using the real --ui-* tokens. The CSS architecture is also the strongest idea here: switch ON/OFF readout and accent are driven entirely by a :has() rule, so the label can never diverge from the checkbox even with JS dead — which is exactly what happened, and it's the only reason the toggles still look alive.

Weakness. A var-hoisting bug throws a TypeError at load and kills almost the entire interactive layer. wireToggle(outToggle,...) at line 702 immediately calls applyStrokeState -> isOutline -> selectedFont -> FONTS.filter, but `var FONTS` isn't assigned until line 735, so FONTS is undefined and .filter throws, aborting the IIFE. Verified on a fresh reload (console: 'Uncaught TypeError: Cannot read properties of undefined (reading filter)'): the font picker never opens, ALL face gating is inert (Fill-tab lock/disable, dim-don't-remove, redirect cue, mode-chip update never run), every segmented control is dead (align, thickening mode, fill type), the weight>1 thickeningMode reveal never fires (visible in the Stroke screenshot: weight=3 with no thickening control), and the Curves toggle — the headline 'BLOCKER fixed' — is itself dead because its wireToggle runs after the throw. Two of the three crux conditional rules (thickeningMode reveal, face-based Fill gating) do not execute at all; only the Fit/Size swap and tab switching survive because they're wired before the throwing line.

Design thesis

Organize the (large) control set into a compact in-panel tab/segment bar: Type · Layout · Stroke · Fill. Each tab shows only its relevant controls, eliminating the long scroll and making conditional sections feel intentional. The Fill tab visibly disables itself (with a reason) for built-in stroke faces. A tiny status line shows the current face type (Stroke vs Outline) so the user understands which tabs are live.

Key moves

  • Fixed 4-segment tab bar (Type · Layout · Stroke · Fill) pinned at panel top with a sliding accent fill (Figma-style equal-width segments). Tabs never reorder or vanish — the locked Fill tab stays in place, dimmed to ~50% with a padlock glyph swapped in for its icon, so the bar never shifts width.
  • A mode chip directly under the bar is the single source of truth: 'Face: Outline (Google)' (blue, glowing dot) vs 'Face: Stroke (built-in)' (muted grey). It wires directly to the face source and explains why Stroke/Fill controls are live or locked.
  • Conditional logic is fully wired and feels intentional: toggling Fit-to-Frame swaps Frame-Fill↔Size in place (no jump); Outline weight >1 reveals Thickening mode via a grid-template-rows reveal that reserves space instead of reflowing; switching to a built-in face dims (never removes) Bezier/Smoothness with an inline reason and locks the whole Fill tab with a 'Fill needs an outline font' note plus a jump-to-Type link.
  • Three cross-cutting Common controls (Smoothing / Curves / Simplify) live in a persistent footer pinned below the scroll area, tagged 'all tabs', so they're reachable from every tab and never buried — directly addressing the researcher's anti-tab-hunting pitfall.
  • Inside each tab, Resolve-style heavy accordions are flattened into quiet uppercase sub-headers with a 3px accent tick, keeping the dense pro-tool feel while grouping Layout, Stroke and Fill logically.
  • Extended fill params (the ~20 shared Fill-engine knobs) are represented compactly: a 4-up icon swatch grid for fill type plus an 'Advanced fill · 21 params' progressive-disclosure drawer, so the section exists and is gated without dumping noise into the main scroll.

Why users love it

A power user opening the Text panel never has to scroll a wall of conditional controls or wonder which ones are live: four equal-width tabs (Type · Layout · Stroke · Fill) with a sliding accent thumb split the large control set into intentional, scannable groups, and a single mode chip plus the Fill tab's padlock tell you at a glance whether you're in Outline or Stroke world. The thesis is now delivered honestly — every claim matches the code. The three sacred Common controls (Smoothing, Curves, Simplify) are pinned to a footer reachable from every tab, and Curves finally tells the truth about its own state because all switches read off the checkbox via CSS :has(). The Fill tab dims rather than vanishes, so even built-in-face users can study what they're missing, and a single prioritized reason (never stacked, never near-invisible) explains exactly why something is paused. One real searchable font picker — five built-in single-stroke faces and a Google list — is the sole input that drives the badge, chip, and gating, so there's one source of truth instead of four mirrors. It feels like a real Vectura panel: 290px, Meridian Blue tokens, mono numerics, accent halos, smooth in-flow reveals — and when the app moves you off a locked tab, it flashes and tells you why instead of yanking the rug.

Adversarial review

Lost functionality flagged: Footer 'Curves' common toggle is never wired (no id, no wireToggle call). Clicking it slides the thumb via CSS :checked, but the 'ON' label text and the accent .sw.on state are driven only by JS — which never runs for this toggle — so the control reads 'ON' permanently while the thumb sits in the off position. The state is unreadable and the value is effectively unsettable/uncommunicated. Functionally the Curves common control is present in markup but broken as a control.; Built-in face hides the ENTIRE Fill tab body via display:none on #fillBody, including fillType/density/angle/advanced — which is per-spec for built-in, but combined with the auto-redirect away from the Fill tab it means a built-in user can never even *see* the fill controls to understand them; only the lock note remains. This is intended dimming taken to removal (display:none), contradicting the team's own 'dim, don't remove' rule for the Fill tab interior.; Smoothness slider (Google-only) sits inside #curveQuality which gets .is-disabled (pointer-events:none) for built-in faces, AND inside #strokeBody which gets .is-disabled when the outline-stroke toggle is OFF. With both conditions true the two opacities (0.4 × 0.4 ≈ 0.16) stack, rendering Bezier/Smoothness near-invisible and giving no single clear reason — the curveLock note is the only legible cue and it only explains the face reason, not the stroke-off reason.

The tab split is a genuinely reasonable answer to the long-scroll pain and most controls are nominally reachable, but the execution does not deliver the surprise-and-delight the thesis promised and breaks on contact with an adversarial user. The single worst problem is a hard functional bug: one of the three sacred 'Common' controls (Curves) is completely unwired, so it lies about its own state — the exact opposite of the 'never wonder what state you're in' selling point. The design's central claims are repeatedly overstated versus what the code does: the 'single source of truth' mode chip is actually one of four redundant face widgets; the 'reserves space, no reflow' thickening reveal does reflow; the 'in-place, no jump' Fit/Size swap uses the very display:none pattern the repo bans and visibly jumps; the 'dim don't remove' Fill rule is actually display:none removal plus a silent forced tab redirect; and the 'Advanced · 21 params' drawer is a placeholder with self-contradicting arithmetic. Accessibility for the locked tab is broken (focusable dead button, no aria-disabled). Visually it is a competent generic dark dashboard that invents its own tokens and Google-hosted fonts rather than committing to Meridian Blue, so it fails the consistency-with-Vectura bar regardless of polish. Net: a promising IA skeleton wrapped in claims the implementation does not honor, with at least one blocker and several majors that must land before this is shippable.
