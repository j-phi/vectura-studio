# Vectura Studio — Comprehensive Audit (2026-05-20, v1.1.10)

Six parallel auditors (architecture, bugs/risk, redundancy, docs, tests, CSS) reviewed the repo after the Meridian cleanup chain closed in v1.1.10. This document consolidates their findings, deduplicates overlap, and prioritizes the work. Each item cites file:line evidence and a suggested direction — not a full fix.

Scope: 333 files, ~177 specs, post-Meridian state (legacy `_ui-legacy.js` and root `styles.css` both deleted; `--color-*` → `--ui-*` token migration claimed complete).

---

## Top 10 highest-leverage improvements (ranked)

1. **Classic-* skin parity is broken (likely live visual bug).** `classic-{dark,light,lark}.css` define only 20 tokens each vs 76 in each `meridian-*.css`. Missing `--render-*`/`--designer-*`/`--vectura-*-rgb` fall through to `components.css:7960–8006` `:root` defaults — which are seeded from **meridian-dark** values. Switching to `classic-light` paints the canvas and designer with dark-mode workspace colors. See §CSS-7.
2. **18 of 20 `src/ui/components/*.js` files are not loaded by `index.html` in production.** Only `info-badge.js` and `empty-state-illustrations.js` are wired up; the rest (Slider, NumStep, Section, Tabs, AngleDial, etc.) are tested via a helper but no production code consumes them. Either dead code (delete) or staged code (wire one panel through them to prove the contract). See §Redundancy-11.1.
3. **Stored XSS in pen-menu rendering from a `.vectura` file.** `src/ui/panels/layers-panel.js:85–89` interpolates `pen.id`, `pen.color`, `pen.width` into `innerHTML` unescaped. A crafted project file plants persistent script execution via cookie-persisted pen records. High severity. See §Bugs-1.
4. **Stored XSS via project-pattern SVG import.** `src/core/pattern-registry.js` `replaceProjectPatterns` accepts attacker SVG from `.vectura` and bypasses `SvgSanitize.sanitize` (which is correctly used elsewhere). Pattern markup later mounts via `innerHTML`. See §Bugs-2.
5. **`renderer.js` is a 6,733-line god module.** Mixes 8+ subsystems (7 tool state machines, touch gestures, selection variants, snapping, geometry helpers, cursor, paint-bucket, light source). Natural seams are obvious; a multi-PR extraction unlocks per-tool tests and ends the "every refactor touches renderer.js" treadmill. See §Arch-1.
6. **Compile-test glut.** 36 of 119 unit tests (~30%) are `*-compile.test.js` files that each spin up their own JSDOM to assert "the file parses and `Foo.bind` exists." Coverage is duplicated by `tests/integration/ui-bootstrap-panels.test.js`. Retire en masse; reinvest in renderer/undo-redo coverage. See §Tests-Red-1.
7. **`.vectura` import has no validation and no rollback.** Numeric params are copied via `JSON.parse(JSON.stringify(...))` with no `Number.isFinite` checks (NaN propagation). Snapshot apply is non-transactional: `applyState` mutates the engine + SETTINGS, then clears history; mid-import failure strands a half-loaded project with no undo escape. See §Bugs-9, §Bugs-10.
8. **`--color-*` and `styles.css` documentation lies.** README, AGENTS.md, CLAUDE.md, docs/skin-authoring.md, docs/a11y-audit-phase5.md still describe `styles.css` as a living legacy sheet being drained; it's been deleted. Three `meridian-*.css` files still declare ~18 `--color-*` aliases each (54 dead declarations) with one orphan reader at `src/ui/ui.js:1000`. See §Docs-multiple, §CSS-2, §Redundancy-5.3.
9. **CHANGELOG and README release notes stop at v1.1.0** while package.json/version.js/badge are at v1.1.10. The entire Meridian cleanup chain (Units 1.5–4.1) is undocumented in user-facing release notes. See §Docs-CHANGELOG.
10. **README algorithm table omits 3 shipped algorithms.** Horizon, Pattern, SVG Distort exist in `src/core/algorithms/` but aren't listed in the README table (in-app help is correct). Prose count is "14+", actual registry is 19. See §Docs-README.

## Architecture

### Arch-1 — Renderer (6,733 LOC) is a god object with 227 graph edges
**Evidence:** `src/render/renderer.js`. Hot methods: `down(e)` (3448–3861, 413 LOC), `move(e)` (3861–4300, 440 LOC), `up(e)` (4300–4562, 263 LOC), `draw()` (2537–3202, 666 LOC). Mixes: tool/cursor state machine, pen tool, shape drafting, algo-draw drafting, direct selection/anchor editing, touch gestures, paint-bucket/fill, snapping/guides, pure geometry helpers, light-source mode, fill loupe.
**Direction:** Extract along natural seams. Keep `Renderer` as canvas+camera+draw loop (~1500 LOC). Move tools into `src/render/tools/{pen,shape,algoDraft,direct,scissor,paintBucket,lightSource}.js` with a registry so `down/move/up` become polymorphic dispatch. Pull pure geometry (`distanceToSegmentSq`, `pointInPoly`, `segmentsIntersect`, `ellipseToPoly`, `circleIntersectsRect`, ~1000 LOC at 5439–6524) into `src/render/geom/intersect.js`, testable without canvas. Move touch state machine into `src/render/input/touch.js`.

### Arch-2 — `renderer → app.ui.*` upward coupling
**Evidence:** `renderer.js:3409, 3415, 5139, 5143, 5145, 5189, 5264` call `this.app.ui.setPaintBucketHint(...)` and `this.app.ui.renderLayers()`. Renderer reaches into UI panel layout.
**Direction:** Add a small event hook on `Renderer` (`renderer.on('paintBucketHint', fn)`); App wires UI listeners.

### Arch-3 — `SETTINGS` is global mutable state, read directly 58× in renderer, 24× in engine
**Direction:** Long-term, pass a frozen `renderConfig` snapshot per draw, or formalize SETTINGS as a typed observable with documented schema. Short-term: document the implicit schema in one place.

### Arch-4 — Two parallel selection systems on `Renderer`
**Evidence:** Layer selection (`selectedLayerIds: Set`, `setSelection`, `clearSelection`, lines 370, 5003–5052) **and** path-anchor "direct" selection (`directSelection`, `startDirectDrag`, `_applyDirectLasso`, lines 413, 1839–2155). Share marquee/lasso/hit-test concepts but separate state, separate code paths.
**Direction:** Introduce `SelectionModel` with two scopes (`layer`, `pathAnchors`) sharing marquee/lasso/hit primitives. Renderer becomes a consumer.

### Arch-5 — Three cooperative cursor-update entry points
**Evidence:** `updateCursor()` (816), `updateHoverCursor(e)` (6577), `_applyModifierCursorOverride()` (6559) cooperatively own `canvas.style.cursor`. Ordering between the three is implicit.
**Direction:** Single `recomputeCursor({event, tool, hover, modifiers})` to eliminate latent ordering bugs.

### Arch-6 — `window.Vectura.ShapeUtils` declared but never consumed
**Evidence:** `renderer.js:6718` exports five members; `grep -r "ShapeUtils" src/ tests/` returns only the declaration.
**Direction:** Delete the export or move helpers into a private module-scope object.

### Arch-7 — Noise Rack convergence is incomplete
**Evidence:**
- `src/core/algorithms/petalis.js:50–142` ships a full `fallbackNoiseRack` with `combineBlend` and `createEvaluator` inline; uses `window.Vectura?.NoiseRack || fallbackNoiseRack`. A parallel implementation people will edit instead of NoiseRack itself.
- `src/core/algorithms/rainfall.js:725, 727` bypasses the rack with direct `noise.noise2D(...)` calls for trail wobble/gust, while the main flow at 131, 227, 283 routes through the rack — two noise spaces in one algorithm.

CLAUDE.md: "new algorithm noise work must converge on the shared model — do not introduce algorithm-specific noise stacks."
**Direction:** Delete the fallback in petalis (load order in `index.html` already guarantees NoiseRack is present); route rainfall's gust/turbulence through a rack evaluator.

### Arch-8 — Hardcoded interaction constants violate config-discipline rule
**Evidence (renderer.js):** `loupe.width = 96` (712), `snapTol = 5/scale` (1201, 2097, 4197), `snapTol = 3/scale` (2350 — different value, no name), `snap = 15` (4145), `handleSize = 6/scale` (5939), `handleR = 2.2/scale` (6035), `snapScissorAngle stepDeg = 15` (1070).
CLAUDE.md: "never hardcode in UI or engine — put in config."
**Direction:** Add `src/config/interaction-tokens.js` with named constants (`SNAP_TOL_PX`, `HANDLE_SIZE_PX`, `SCISSOR_ANGLE_STEP_DEG`, etc.).

## Bugs & risk

### Bugs-1 — HIGH: Stored XSS via `.vectura` pen records
**Evidence:** `src/ui/panels/layers-panel.js:85–89` interpolates `pen.id`, `pen.color`, `pen.width` into `innerHTML` without escaping. Only `pen.name` runs through `escapeHtml`. `SETTINGS.pens` is restored verbatim from imported `.vectura` (`src/app/app.js:593–595`, `src/ui/ui-file-io.js:112`) with no validation.
**Scenario:** A crafted file with `pen.color = '"><img src=x onerror=alert(1)>'` fires next time the pen menu opens; persists via cookie/localStorage.
**Direction:** Escape every interpolated field, or rebuild the menu via `createElement` + `textContent` + `.style.background = pen.color`. Validate pen records on import (id pattern, color hex regex, width finite).

### Bugs-2 — HIGH: Stored XSS via project-pattern SVG
**Evidence:** `src/core/pattern-registry.js:43–64` `normalizePattern` does `svg: \`${pattern.svg || ''}\`` without sanitization. `replaceProjectPatterns` is invoked from the `.vectura` open path (`src/ui/ui-pattern-designer.js:988–989`). Pattern markup later mounts via `tempSvg.innerHTML = svg.innerHTML` at `src/core/algorithms/pattern.js:1499`. `<script>` in `innerHTML` doesn't execute, but `onerror`/`onload` on `<image>`/`<a>` and `<animate attributeName="href" to="javascript:...">` do.
**Direction:** Run `Vectura.SvgSanitize.sanitize` on `pattern.svg` in `replaceProjectPatterns` and `upsertIntoPool('project', …)`. Add a security test for this path.

### Bugs-3 — MEDIUM: Pen color/width drags push no undo history
**Evidence:** `src/ui/panels/pens-panel.js:285–298, 302–314`. `colorInput.oninput`/`widthInput.oninput` mutate state and propagate without `pushHistory`. No `onchange` handler.
**Direction:** Snapshot on drag-start; push on `change` commit. Mirror the transform-commit pattern at `app.js:111`.

### Bugs-4 — MEDIUM: `openModal()` legacy path injects body as innerHTML, no focus trap, no Esc
**Evidence:** `src/ui/overlays/modal.js:216–236`. `this.modal.bodyEl.innerHTML = body` for string bodies; no Esc handler, no `focus.trap` call. Used by error toasts in `ui-file-io.js:117–122`.
**Direction:** Require `body` be a `Node` (drop the string branch). Add Esc keydown + `focus.trap(card)` on open.

### Bugs-5 — MEDIUM: SVG sanitizer doesn't drop external `href`/`xlink:href`, `<animate to="javascript:...">`, `<use href="…">`
**Evidence:** `src/core/svg-sanitize.js` removes `<script>` and `on*` handlers but not external resource refs or animation-href attacks. `parseSvgToLayerGroups` mounts cloned SVG into the live document to call `getCTM`/`getTotalLength`.
**Direction:** Strip `href`/`xlink:href` not starting with `#` (or allowlist); drop `<animate>`/`<set>` whose `attributeName` is `href` or whose `to`/`from`/`values` start with `javascript:`; drop `<use>` with non-fragment refs.

### Bugs-6 — MEDIUM: `pattern-registry` silently swallows localStorage errors
**Evidence:** `src/core/pattern-registry.js:85–91` `catch (err) {}`. `hydrateLocalPatterns` (148–160) resets to `[]` silently on parse failure. `pattern.js` already enforces `console.warn` per the security test; pattern-registry was missed.
**Direction:** Add `console.warn('[PatternRegistry] ...', err)` in both catch blocks.

### Bugs-7 — MEDIUM: Preference snapshot applied without per-field validation
**Evidence:** `src/app/app.js:426–445`. `JSON.parse` is guarded but `applyPreferenceSnapshot` (314–424) uses `??` fallback without numeric/string validation. `SETTINGS.bgColor` from snapshot flows into inline `style.background`.
**Direction:** Validate each field — colors via hex regex, numbers via `Number.isFinite` + clamp, enums via allowlist (already done for `paneLeftWidth`).

### Bugs-8 — MEDIUM: Numeric ranges not validated on `.vectura` import; NaN propagation
**Evidence:** `src/core/engine.js:354–355` `JSON.parse(JSON.stringify(data.params || {}))`. Algorithms divide by `params.scaleX`, `params.density` etc. without `Number.isFinite` guards.
**Direction:** `Number.isFinite` checks + clamps when copying numeric params on import, mirroring the `strokeWidth` handling at line 398.

### Bugs-9 — MEDIUM: Non-transactional `.vectura` import strands half-loaded projects
**Evidence:** `src/ui/ui-file-io.js:112–114`, `src/ui/ui-pattern-designer.js:1002–1004`. `applyState` mutates `engine.layers` and SETTINGS in place; history is cleared after. If `engine.importState` throws mid-flight, the project is half-loaded and unrecoverable.
**Direction:** Snapshot before any mutation; attempt the import inside try; roll back via `applyState(snapshot)` on failure. Only clear history once new state is fully installed.

### Bugs-10 — LOW: `exportSVG` leaks object URLs
**Evidence:** `src/ui/ui-file-io.js:646–651` creates `URL.createObjectURL` but never revokes (contrast `saveVecturaFile:76–84`).
**Direction:** `URL.revokeObjectURL(url)` after `a.click()`.

### Bugs-11 — LOW: `joinNearbyPaths` outer `while(changed)` has no iteration cap
**Evidence:** `src/core/optimization-utils.js:115–141`. Guaranteed-terminating but adversarial input could degrade quadratically.
**Direction:** Add `MAX_ITER = source.length * 4` cap with warn-and-bail.

### Bugs-12 — LOW: `regen()` mutates engine output without pushing history
**Evidence:** `src/app/app.js:808–816`. Generation-only changes silently break the redo stack.
**Direction:** Either clarify `regen()` is display-only, or have it `pushHistory` if `isRestoring` is false and state differs.

### Bugs-13 — LOW: `applyState` race against in-flight async generation
**Evidence:** Some algorithms (image-loading) enqueue `setTimeout`/rAF; rapid open-file → open-file can land stale paths in the new project.
**Direction:** Tag each `generate` with a monotonic engine epoch; discard stale results.

## Redundancy & consistency

### Redundancy-1 — Helper-duplication map (most significant)
- `clamp` — 5 distinct definitions + 42 inline `Math.max(min, Math.min(max, …))` sites across src. Canonical: `AlgorithmUtils.clamp` (`src/core/algorithm-utils.js:15`); local copies at `ui/utils.js:17` (NaN-handling variant), `ui/ui.js:288` (`clampPointToRect`), `ui/shell/toolbar.js:661` (`clampFloat`).
- `clamp01`, `lerp` — duplicated between `algorithm-utils.js` and `geometry-utils.js` (identical).
- `roundToStep` — three identical copies in `ui.js:217`, `randomization-utils.js:12`, `panels/noise-rack-panel.js:48`.
- `escapeHtml` — two definitions (`ui.js:191`, `ui-pattern-designer.js:10`).
- `clone` (deep clone) — four IIFE-local definitions; 51 `JSON.parse(JSON.stringify(...))` calls vs 5 `structuredClone` uses.
- `pointInPolygon` — `path-boolean.js:31` (canonical) and `algorithms/lissajous.js:99` (local).
- `smoothPath` — `geometry-utils.js:5` (canonical) is byte-identically duplicated at `ui.js:754`; `algorithms/topo.js:176` is a different `chaikinSmooth`-style variant.
- `isClosedPath` — canonical export shadowed by 5 `(() => false)` fallbacks (in engine, masking, pathfinder-ops, algorithms/pattern). Fallbacks hide load-order bugs.
- `frac` — `AlgorithmUtils.frac` shadowed by locals in petalis, rings (×2), ui-petal-designer.
- `getEl` — defined four times (ui.js, ui-touch.js, ui-document-units.js, panels/algo-config-panel.js).
- Color conversions — `renderer.js:855` canonical; `color-picker.js:86–104` has a full self-contained set; multiple `toHex` locals in panels and export modal.
**Direction:** Promote canonical helpers; delete duplicates. Highest payoff: consolidate `clone`, `clamp`, `smoothPath`, `isClosedPath`, `escapeHtml`. Track explicit "NaN-tolerant clamp" variants if intentional.

### Redundancy-2 — Two `window.Vectura` namespace-init styles
**Evidence:** Older form `window.Vectura = window.Vectura || {};` in 30+ files. Newer Phase-1 form `const Vectura = (window.Vectura = window.Vectura || {});` in 15+ files. `src/ui/shortcuts.js:23` and `persistence.js:23` reference `Vectura.UI` without ensuring `Vectura` exists — fragile under script-reorder.
**Direction:** Standardize on the newer form; fix the two fragile callers.

### Redundancy-3 — `path-boolean.js` and `fill-boolean.js` have overlapping ring-normalization stacks
**Evidence:** `path-boolean.js:153` + `fill-boolean.js:181`. `pathfinder-ops.js` delegates to FillBoolean's `ringToMultiPolygon`/`rectToMultiPolygon`/`polygonsToMultiPolygon`. `path-boolean.js` has its own `closePolygonIfNeeded`/`normalizePolygons`.
**Direction:** Audit whether `path-boolean.js`'s ring normalization can be absorbed into `FillBoolean`.

### Redundancy-4 — `components.css` is 9,246 LOC — the new "styles.css"
**Direction:** Split into `components/{panels,toolbar,modals,canvas}.css` before it repeats the prior anti-pattern.

### Redundancy-5 — Stale "Meridian" / `_ui-legacy.js` comments in src
**Evidence:** 6+ stale doc-comments referencing `_ui-legacy.js` in `src/ui/shell/pane-left.js:180`, `shell/toolbar.js:938`, `overlays/modal.js:27,172`, `panels/algorithm-panel.js:88`, plus many in `ui.js` body. Test file `tests/unit/legacy-file-removed.test.js` correctly asserts deletion — but production code comments still reference the dead file.
**Direction:** Single-PR sweep.

### Redundancy-6 — `_UI*Mixin` underscore-prefix idiom is misleading
**Evidence:** `_UIFileIOMixin`, `_UITouchMixin`, `_UIDocumentUnitsMixin`, `_UIPetalDesignerMixin`, `_UIPatternDesignerMixin`, `_UIRandomizationMixin`, `_UIFillPanelMixin`, `_UINoiseDefs`, `_UIExportUtil`. Underscore signals private; 13+ external call sites prove they're public.
**Direction:** Drop the underscore or genuinely encapsulate.

### Redundancy-7 — Production hygiene is otherwise clean
- `grep TODO|FIXME|HACK src/` → 0 hits in first-party code (only 2 in vendored `polygon-clipping.umd.js`).
- 22 production `console.*` calls, all `warn`/`error` with bracketed subsystem prefixes (`[Engine]`, `[UI]`, etc.). No `console.log`, no `debugger;`. Minor: `[applyTheme]` should be `[App]` for prefix consistency.

## Tests

### Tests-Red-1 — Retire the `*-compile.test.js` cluster
**Evidence:** 36 of 119 unit tests; ~4,000 LOC; ~25 of them inline their own `new JSDOM(...)` rather than using `tests/helpers/load-vectura-runtime.js`. Each asserts roughly: `Vectura.UI.Foo.bind exists` + `throws before bind` + `does not throw with sentinel deps`. The same regression is caught by `tests/integration/ui-bootstrap-panels.test.js`, which boots the real shell.
**Direction:** Replace with one consolidated "expected UI surface area" smoke test; keep dep-bag negative tests only for satellites where wiring is genuinely fragile. Expected savings: ~4,000 LOC, ~25 JSDOM cold-starts per `npm run test:unit`.

### Tests-Red-2 — Skipped suites and gated files
- `tests/unit/horizon.test.js:84` has `describe.skip('Horizon algorithm', ...)` with 80+ LOC of dead fixtures above it. Per visual-baselines git log, Horizon was hidden then revived — the suite was never re-enabled.
- `tests/e2e/mask-shift-drag.spec.js` is 186 LOC where the only `test()` is `test.skip(...)` (tablet-touch synthesis problem).
- `tests/e2e/visual.spec.js` is entirely gated behind `ENABLE_SCREENSHOT_VISUALS=1` (line 6). Snapshots exist, so the file did run at some point.
- Five `test.skip(testInfo.project.name.includes('tablet-touch'), ...)` in `tests/e2e/smoke.spec.js`, all with the same justification — skips are normalized rather than tracked.

**Direction:** Decide and act. Each skip should either be unblocked or deleted. File a single tracking issue for tablet-touch synthesis and link from each skip.

### Tests-Red-3 — `mask-preview.test.js` exists in both unit/ and integration/
**Evidence:** Both files exercise `getMaskPreviewClipPolygons`/`startMaskPreview` against a real `Renderer`. The "unit" file is also effectively an integration test.
**Direction:** Merge into the integration file.

### Tests-Gap-1 — Renderer (6,733 LOC) has ~8 unit specs
**Direction:** Move pure geometry out of `Renderer` (see Arch-1) and add unit specs for snapping resolvers, transform-handle hit-test, pointer→world conversion, multi-touch pinch math.

### Tests-Gap-2 — Undo/redo has one integration test (layer structure only)
**Direction:** Add unit specs per reversible operation (set param → undo restores), parameterized over the operation registry.

### Tests-Gap-3 — Per-algorithm noise-rack convergence is uncovered
**Evidence:** `noise-rack.test.js` is 195 LOC of unit tests; no per-algorithm integration verifying every algorithm consumes the rack identically (the convergence contract).
**Direction:** Parameterized test iterating every registered algorithm: "feeding a fixed rack produces deterministic output, no algorithm bypasses to raw `noise.noise2D`."

### Tests-Gap-4 — Visual baselines cover half the algorithm registry
**Evidence:** 16 SVG baselines, 7 PNG snapshots vs 19 registered algorithms. Missing: boids, attractors, hyphae, harmonograph, topo, grid, phylla, spiral.
**Direction:** One canonical SVG baseline per algorithm; extend `npm run test:update` to auto-generate the missing ones.

### Tests-Gap-5 — E2E lacks round-trip and SVG-import journeys
**Evidence:** No Playwright test for: load `.vectura` → mutate → save → reopen → identical; SVG import via `setInputFiles` followed by render check.
**Direction:** One Playwright spec each.

### Tests-Gap-6 — XSS / SVG-sanitization corpus is thin
**Evidence:** `tests/unit/security_xss.test.js` (189 LOC), `tests/integration/export-sanitization.test.js` (102 LOC).
**Direction:** Add an attack corpus fixture (one file per known vector class — `<animate to="javascript:...">`, external `<image href>`, `<use href>` to remote, `foreignObject` with HTML, CSS `url()` etc.) and run sanitization against each. Cover the project-pattern path (currently uncovered — see Bugs-2).

### Tests-Brittleness-1 — 44 lines of hex-literal assertions in unit tests
**Evidence:** `grep -E "#[0-9a-fA-F]{6}" tests/unit/*.test.js | wc -l → 44`. Likely pen palette / theme defaults.
**Direction:** Replace literal assertions with `expect(...).toBe(getDefaultPenColor())`-style references.

### Tests-Discipline — Clean elsewhere
- No `.only`, no `xit`, no `xdescribe`.
- `.test.js` vs `.spec.js` convention is consistent (unit/integration use `.test.js`; e2e uses `.spec.js`).
- `tests/helpers/load-vectura-runtime.js` adoption is high (70 tests) outside the compile-test cluster.
- `legacy-file-removed.test.js` is an exemplary "tombstone test." Consider applying the same pattern to `styles.css` ("does not grow past current LOC").

### Tests-Other — Helper file is suspicious
**Evidence:** `tests/helpers/vitest.setup.js` is 1 LOC.
**Direction:** Either delete it, or lift the `bootErrors` listener pattern from `ui-bootstrap-panels.test.js` into global setup.

## CSS / skin system

### CSS-1 — Classic-* skin parity is broken (likely live visual bug)
**Evidence:** Token counts — `meridian-{dark,light,lark}.css`: 76 each. `classic-{dark,light,lark}.css`: **20 each**. Missing from classic-*: `--render-*` (15), `--designer-*` (10), `--plotter-*` (3), `--vectura-*-rgb` (7), `--color-*` aliases (18). These fall through to `components.css:7960–8006` `:root` defaults — which are **meridian-dark** values. Switching to `classic-light` paints canvas/designer with dark-mode workspace colors.
**Direction:** Define `--render-*`, `--designer-*`, `--plotter-*`, `--vectura-*-rgb` per classic-* skin, or remove the `:root` block from components.css and require every skin to define them.

### CSS-2 — `--color-*` aliases are dead in three meridian-*.css files (54 declarations)
**Evidence:** `meridian-{dark,light,lark}.css:47–65` each declare ~18 `--color-*` aliases. Only one production reader remains: `src/ui/ui.js:1000` (`getThemeToken('--color-accent', '#fafafa')`). Renderer's `getThemeToken` (`renderer.js:77`) has a `name === '--color-accent'` branch but is no longer called with that key elsewhere.
**Direction:** Step 3.3b — migrate `ui.js:1000` to `--ui-accent`; delete alias blocks in all three meridian-*.css; delete the renderer branch.

### CSS-3 — `components.css:7960–8006` ships its own `:root` token block (skin-fallback hack)
**Evidence:** Declares `--vectura-*-rgb`, `--render-*`, `--designer-*`, `--plotter-*` at `:root` — a fallback for classic-* skins that don't define them.
**Direction:** This is the root cause of CSS-1. Move tokens into each skin (or into `tokens.css` if truly skin-agnostic).

### CSS-4 — `[data-ui-skin]` prefix on 353 selectors in components.css
**Evidence:** Many `[data-ui-skin] .foo {}` selectors pin an attribute that has no value match — purely a specificity wrapper, likely defensive against the deleted `styles.css`.
**Direction:** Drop the prefix where it doesn't gate on a skin value. Likely cuts `!important` count from 29 to ~10 once specificity fights stop.

### CSS-5 — 30+ inline hardcoded colors inside selectors
**Evidence:** `components.css:138 .hdr-btn.is-danger { color: #fff }`, `:299 .seg-opt.active { color: #fff }`, `:801 background: #fff`, `:2164 color: #c4a8e0`, `:2180 color: #d8b4fe`, `:6583–6587 .sect--color-* { --sect-bar: #A78BFA; ... }`.
**Direction:** Introduce `--ui-on-accent`, `--ui-on-danger`; replace literals.

### CSS-6 — `components.css` is 9,246 LOC (the new monolith)
See Redundancy-4.

### CSS-7 — Doc says transitions go in motion.css; reality is 126 transition declarations in components.css
**Evidence:** `motion.css: 0` transitions, `components.css: 126`. 10 of the 126 are `transition: all 0.12s` — animates every property including box-shadow and transform.
**Direction:** Either update CLAUDE.md ("component transitions stay co-located; motion.css owns keyframes") or extract. At minimum, replace `transition: all` with explicit property lists in the 10 hot spots.

### CSS-8 — Five panel-level `@media` rules are container-query candidates
**Evidence:** `components.css:5378, 5616 (1100px)`, `:6984 (1024px)`, `:7027 (900px)` target left/right pane behavior. Per CLAUDE.md "prefer container queries over viewport media queries for panel/component-level responsiveness."
**Direction:** Set `container-type` on the pane shell; convert these four to `@container`. Mobile breakpoints (`540px`, `640px`, `pointer:coarse`) legitimately stay as `@media`.

### CSS-9 — `classic-light` `--ui-muted: #888888` on `--ui-bg: #f3f4f6` is borderline AA (~3.6:1)
**Direction:** Verify with a contrast checker; bump to `#6b7280` or darker if AA-fail.

### CSS-10 — 11 inline `style="display:none"` on `<div>` initial states in index.html
